# 宏图作业指导书生成器 · 验收标准

每个里程碑必须**全部条目通过**才算交付。验收人：用户本人。

---

## M1：CLI 核心

### 功能验收

- [ ] **F1.1** `python gen.py products/DEMO01.yaml` 在 `output/` 下生成 `DEMO01.html`，浏览器打开后**视觉一致**于当前手写版（封面、目录、流程图、8 道工序页）。
- [ ] **F1.2** `python gen.py products/DEMO01.yaml --pdf` 同时生成 `output/DEMO01.pdf`，工序页正确横向显示，**没有任何工序跨页**。
- [ ] **F1.3** `python gen.py products/DEMO01.yaml --check` 仅校验、不渲染，校验通过返回 0，失败返回非 0 + 中文错误信息。
- [ ] **F1.4** `python gen.py products/*.yaml --pdf` 支持批量（用通配符或多个文件参数）。

### 数据/模板验收

- [ ] **D1.1** `products/DEMO01.yaml` 内容完整复刻当前手写 HTML 中所有 8 道工序的：工序名、关键工序标记、操作说明、注意事项、工具、材料、图片。
- [ ] **D1.2** `products/_schema.yaml` 包含字段说明 + 字段约束 + 一个完整示例。
- [ ] **D1.3** 模板拆分清晰：`cover` / `toc` / `flow` / `process` 四个分片，互不依赖产品名等硬编码。

### 校验验收

- [ ] **V1.1** 故意写一条 `operations` 长度超过 30 字 → 校验给出警告。
- [ ] **V1.2** 故意写 7 条 `operations` → 校验报错，渲染中止。
- [ ] **V1.3** 故意写 `images: [missing.png]` 引用不存在文件 → 校验报错。
- [ ] **V1.4** 故意把 `processes[].images` 写成 3 张 → 校验报错。

### 工程验收

- [ ] **E1.1** `requirements.txt` 列出依赖（jinja2、pyyaml），其余用 Python 标准库。
- [ ] **E1.2** `README.md` 包含：项目简介、安装、CLI 用法、YAML 字段说明、常见问题。
- [ ] **E1.3** 与当前手写 HTML 对比 PDF（DEMO01），逐页**视觉一致**（差异 ≤ 像素级抖动）。

## M2：Claude Code Skill

### 功能验收

- [ ] **S2.1** `.claude/skills/doc-gen/SKILL.md` 存在，被 Claude Code 识别为可用 Skill。
- [ ] **S2.2** 描述场景"参考 DEMO01，给我生成 DEMO02，工序如下：[列出 5 道工序]"，Claude 能：
  - 自动复制 `products/DEMO01.yaml` 为 `products/DEMO02.yaml`
  - 按描述填入工序内容
  - **严格遵守字段约束**（不超长、不超数量）
  - 调用 `python gen.py products/DEMO02.yaml --pdf`
  - 报告输出位置
- [ ] **S2.3** 描述场景"给 DEMO01 的工序 3 加一条注意事项：XXX"，Claude 能精确修改 YAML 并重新生成。

### 文档验收

- [ ] **S2.4** SKILL.md 包含：触发关键字、YAML schema 摘要、字段约束、典型工作流、常见错误处理。
- [ ] **S2.5** SKILL.md 长度 ≤ 150 行（避免 token 浪费）。

## M3：PySide6 GUI

### 功能验收

- [ ] **G3.1** 启动 `python -m gui.main`，主窗口显示：左侧产品/工序树、中间编辑表单、右侧 HTML 实时预览。
- [ ] **G3.2** 新建产品：点"新建" → 输入型号 → 自动创建空 YAML + 图片目录。
- [ ] **G3.3** 工序编辑：所有字段（工序名、关键标记、操作说明、注意事项、工具、材料、图片）均可编辑，字段超长红色高亮。
- [ ] **G3.4** 工序拖拽排序生效，保存后 YAML 顺序正确。
- [ ] **G3.5** 图片拖入：拖入图片自动复制到 `assets/images/<model>/`，自动填入 `images:` 字段。
- [ ] **G3.6** 实时预览：编辑任何字段，预览 1 秒内刷新。
- [ ] **G3.7** 工具栏"导出 PDF" → 调 Edge headless → 弹出"完成"对话框，含输出路径。
- [ ] **G3.8** 工具栏"复制为新产品" → 弹出对话框输入新型号 → 派生新 YAML（含所有工序、不含图片，提示用户重新选图）。

### 易用性验收

- [ ] **G3.9** 非技术使用者（工艺员）在 10 分钟内可独立完成"新建产品 → 添加 3 道工序 → 导出 PDF"全流程，无需阅读文档。
- [ ] **G3.10** 任何字段输入错误（超长、为空、文件缺失），界面给出**中文提示**和具体定位。

### 工程验收

- [ ] **G3.11** GUI 启动 ≤ 2 秒。
- [ ] **G3.12** 大批量产品（10+ 个）时，左侧树仍流畅。

## 跨阶段通用验收

- [ ] **C1** 全流程**不依赖**任何在线服务（包括 Claude API）；离线机器可完成所有 M1/M3 功能。
- [ ] **C2** 文件编码统一 UTF-8（含 YAML、模板、Python 源码）。
- [ ] **C3** 所有 Python 模块通过 `python -m py_compile` 编译检查。
- [ ] **C4** 公司名等"项目级常量"集中在 YAML 或一个 `core/constants.py` 中，**不在模板里硬编码**。
- [ ] **C5** Git 提交粒度合理（每个里程碑至少一个独立 commit，commit message 中文遵循 `type(scope): 描述`）。
