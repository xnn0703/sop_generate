sop_generate 使用说明

公司内部标准化作业指导书（SOP）生成工具。把工艺步骤填进表单，自动产出 A3 排版的 HTML 和 PDF 文档。

📖 本文档章节

  • 软件能解决什么问题
  • 如何安装到 Windows / Mac
  • 5 分钟做完第一份 SOP
  • 怎么编辑工序、加图、排序
  • 怎么导出 PDF 拿去打印
  • 怎么自动接收新版本
  • 遇到问题怎么办

1. 软件简介

1.1 它能做什么

把工艺员脑子里的"装配步骤"快速变成规范、统一、可打印的作业指导书。

  • 📄 统一排版：所有产品的 SOP 共用同一套 A3 模板，封面、目录、工艺流程图、工序详情页全部自动生成
  • 🖼️ 支持多图：每道工序可以放 1-4 张图，自动 1 张全占 / 2 张上下 / 3-4 张 2×2 网格布局
  • ⚙️ 所见即所得：左边填字段，右边实时预览，改完直接看效果
  • 💾 导出 HTML + PDF：内部归档用 HTML，打印发车间用 PDF
  • 🔄 自动更新：启动时检查新版本，一键升级，老数据自动保留

1.2 它不能做什么

  • 不是 Word 替代品：模板是固定的 A3 排版，不支持自由排版
  • 不绘制工序图：图片需要工艺员自己用相机/截图准备
  • 不连接 PLM/ERP 系统：是独立工具，数据本地存

1.3 适合谁用

角色使用方式
工艺员 / 工艺工程师主要用户。用图形界面新建产品、编辑工序、导出 PDF
班长 / 生产主管查看已有 SOP，新版本时一键升级软件
软件工程师可用命令行批量生成；可用 Claude AI 自然语言起草 YAML

2. 安装指南

2.1 Windows 安装

📦 准备工作（只需一次）

  1. 从浏览器下载 7-Zip 并安装。这是解压软件，分发软件需要它
  2. 确认电脑装了 Microsoft Edge（Win10/11 自带，无需额外操作）或者 Google Chrome。没装浏览器的话 PDF 导出会用不了

📥 下载与解压

  1. 打开公司提供的下载链接（Gitee 发版页面），找到最新版本
  2. 下载 Windows 下载 章节下的所有 5 个分卷： 
    sop_generate-win-vX.Y.Z.7z.001
sop_generate-win-vX.Y.Z.7z.002
sop_generate-win-vX.Y.Z.7z.003
sop_generate-win-vX.Y.Z.7z.004
sop_generate-win-vX.Y.Z.7z.005
    必须 5 个全部下载且放在同一个文件夹里，缺一个会解压失败
  3. 在 5 个分卷所在的文件夹里，右键 .7z.001（注意是 001 不是其他）→ 7-Zip → 解压到当前位置
  4. 解压后会出现一个 sop_generate-win/ 文件夹
  5. 建议把这个文件夹移到方便的位置，例如 D:\sop_generate\

🚀 首次启动

  1. 双击文件夹里的 sop_generate.exe
  2. Windows Defender 可能弹窗"未识别的应用"。点 "更多信息" → "仍要运行"
  3. 看到主界面出现，标题栏显示版本号即成功
建议把 sop_generate.exe 在桌面创建快捷方式：右键 .exe → "发送到" → "桌面快捷方式"

2.2 macOS 安装

📦 准备工作（只需一次）

  1. 从浏览器下载 Keka 并安装（macOS 解压软件，免费）
  2. 确认装了 Microsoft Edge 或 Google Chrome（用于 PDF 导出）

📥 下载与解压

  1. 打开公司提供的下载链接（Gitee 发版页面）
  2. 下载 macOS 下载 章节下的所有 4 个分卷
  3. 双击 .7z.001（Keka 自动识别）→ 解压
  4. 得到 sop_generate-mac-vX.Y.Z/ 文件夹，建议放在 ~/Applications/ 或桌面

🚀 首次启动

  1. 双击 sop_generate.app
  2. 系统弹"无法验证开发者"或"已损坏" → 不要点"移到废纸篓"
  3. 打开 系统设置 → 隐私与安全性 → 滚动到底部，找到 "sop_generate 已被阻止" 行 → 点 "仍要打开"
  4. 再次双击 sop_generate.app 即可正常运行
macOS 的安全机制对未签名应用是这样。这是一次性操作，之后不会再问。

3. 5 分钟上手（做第一份 SOP）

以下是一个最小示例，跟着做下来你就理解全流程了。

  1. 启动软件。左侧产品列表里应该已经有一个示例 DEMO01
  2. 点工具栏 新建产品。弹窗输入产品型号，例如 TEST01，确定
  3. 左侧出现 TEST01。切换到中间的 "产品信息" 标签页，填入：
    • 产品名称：测试产品
    • 公司名：你的公司名
    • 文件编号：DOC-001
    • 发布日期 / 实施日期：填日期
  4. 左下角点 "＋ 工序" 按钮添加一道工序
  5. 切换到 "工序编辑" 标签页，填入：
    • 工序名：测试工序
    • 操作说明：每行一条，例如 "1. 拿起零件"、"2. 装入夹具"
    • 工具设备 / 作业材料：每行一条
  6. 把一张工序图直接拖到右下角"图片"区域，软件会自动复制到产品图片目录
  7. 右侧应该已经显示实时预览，看效果
  8. 点工具栏 保存
  9. 点工具栏 导出 PDF，等几秒 → 弹窗提示路径 → 用文件管理器打开 output/TEST01.pdf 查看
这就是完整流程：新建产品 → 加工序 → 填字段 → 拖图 → 导出 PDF。下面章节是每一步的详细说明。

5. 产品管理

5.1 新建产品

  1. 点工具栏 新建产品
  2. 弹窗输入产品型号，要求：
    • 只能含字母、数字、下划线、横线
    • 不能含中文、空格、特殊符号
    • 例：XESA01、RX-2000、DEMO_v2
  3. 确定后软件自动：
    • 创建空白 YAML 文件 products/<型号>.yaml
    • 创建图片目录 assets/images/<型号>/
    • 添加一道默认工序
  4. 切到 "产品信息" 标签页填写封面字段

5.2 产品信息字段

字段说明示例
产品型号同 YAML 文件名，新建后不能改XESA01
产品名称显示在工序页表头射频终端
公司名显示在封面顶部（绿色字）你公司全称
文件编号显示在封面和每页表头DOC-XX-001
版本形如 A/0、B/1A/0
发布日期YYYY-M-D 格式2026-1-15
实施日期YYYY-M-D 格式2026-2-1

5.3 派生产品（基于已有产品快速建模）

同一系列的产品工序结构往往类似。派生可以复制当前产品的所有工序当作新产品的起点。
  1. 左侧选中作为模板的产品（例：XESA01）
  2. 点工具栏 派生新产品
  3. 输入新型号（例：XESA02）
  4. 软件自动：
    • 复制所有工序结构（工序名、操作说明、注意事项等）
    • 清空图片字段（新产品零件不同，需要重新选图）
    • 创建空的图片目录 assets/images/XESA02/
  5. 在新产品上修改差异部分 + 替换图片即可

5.4 删除产品

软件界面没有删除按钮。如果要删除产品：

  1. 关闭软件
  2. 用文件管理器删除 products/<型号>.yaml
  3. 用文件管理器删除 assets/images/<型号>/ 整个目录
  4. 重新启动软件

6. 工序编辑

6.1 添加工序

  1. 左侧选中产品
  2. 左下角点 ＋ 工序 按钮
  3. 新工序出现在列表最底部，名字默认是"新工序"
  4. 切到 "工序编辑" 标签页修改字段

6.2 删除工序

  1. 左侧选中要删除的工序
  2. 点 － 工序 按钮
  3. 立即生效，不会二次确认。如果误删可以撤回吗？不可以，必须重新填一遍。建议删除前用 保存 备份当前 YAML

6.3 调整工序顺序（拖拽）

  1. 左侧工序列表里，按住鼠标左键拖动工序行
  2. 拖到目标位置，松开
  3. 顺序立即生效，右侧预览自动更新（封面、目录、流程图都跟着重新排列）
  4. 记得点 保存

6.4 标记关键工序 ★

关键工序在文档中会用红色 ★ 标识。涉及静电敏感器件、精密装配等的工序应该标记。

  1. 切到 "工序编辑" 标签页
  2. 勾选 ☐ 关键工序（★）
  3. 保存后会在以下位置出现 ★：
    • 封面页的工序列表
    • 目录页的工序条目
    • 工艺流程图的对应节点
    • 工序详情页表头

7. 字段填写规则

每个工序有 6 类字段。下面说明每类的规则和数量限制。

7.1 操作说明

工序的具体步骤。每行一条，按填写顺序自动加序号 1、2、3...

项目规则
条数上限18 条，超出会校验报错
单页最多6 条；超过 6 条自动拆页，标"续 N/M"
字符长度不强制限制，但建议每条 ≤ 30 个汉字
推荐做法如果超过 6 条，优先把工序拆成两道而不是依赖自动拆页，这样图片和注意事项也能各自配套

7.2 注意事项

每行一条，自动加 ①、②、③、④ 编号。

项目规则
条数上限4 条
字符长度不强制，但建议每条 ≤ 28 个汉字（超长会自动换行）
超出怎么办如果有 5 条注意事项，建议把工序拆成两道

7.3 工具设备 / 作业材料

两个并列字段，分别填工具和材料。每行一条。

项目规则
条数上限每项 4 条
字符长度不强制，但建议 ≤ 10 个汉字。太长会换行后被截断（不影响布局）
示例工具："内六角螺丝刀"、"扭力扳手"；材料："M3×8 螺钉"、"螺纹胶"

7.4 工序名

项目规则
字符长度不强制，但建议 ≤ 12 个汉字。超长会自动换行显示
用途显示在封面工序列表、目录、工艺流程图节点、工序页表头
关于"字符长度无硬限"：早期版本对每个字段长度有严格限制，超长直接报错。现在改成 CSS 自动换行+裁切——你可以填得长一点，软件在单元格里换行显示，溢出部分被裁掉不影响布局。这样工艺员有更大灵活性，但仍建议简洁，因为太长读起来累。

8. 图片管理

8.1 添加图片（拖拽）

  1. 准备好图片文件（推荐 PNG，JPG 也可以）
  2. 从文件管理器拖到中间编辑区右下方的"图片"区域
  3. 软件自动：
    • 把图片复制到 assets/images/<产品型号>/
    • 把文件名填入 images 字段
    • 右侧预览立即显示
如果两个产品的工序使用同一张图，可以在不同产品目录下放同名文件，或者手动改 YAML 的 image_base 路径。普通用户直接拖拽即可。

8.3 图片大小建议

项目建议
分辨率不低于 1280×800，太低打印会糊
格式PNG（推荐）或 JPG
单图体积不要超过 5MB，太大文档体积膨胀

8.4 图片不显示？

  1. 检查图片文件是否真的在 assets/images/<产品型号>/ 下
  2. 检查 YAML 里 images 字段的文件名是否拼写正确（区分大小写、含扩展名）
  3. 试着重新拖拽一次

9. 预览与导出

9.1 实时预览

右侧区域是 HTML 实时预览。任何字段改动后约 0.6 秒自动刷新。

如果觉得没刷新或者卡住，点工具栏 刷新预览 强制刷新。

预览的是 HTML 排版，和导出的 PDF 视觉一致。打印效果直接看右侧就行。

9.2 导出 HTML

  1. 点工具栏 导出 HTML
  2. 软件自动渲染并保存到 output/<产品型号>.html
  3. 弹窗提示完成路径
  4. 用浏览器打开即可

用途：归档、邮件分发、内部网页浏览。

9.3 导出 PDF

  1. 确认电脑装了 Microsoft Edge 或 Google Chrome（首次需要）
  2. 点工具栏 导出 PDF
  3. 软件自动：
    • 先渲染 HTML
    • 调浏览器以"无界面"模式打开 HTML，打印为 PDF
    • 保存到 output/<产品型号>.pdf
  4. 弹窗提示完成路径

用途：打印纸质 SOP 发到车间。

如果导出 PDF 失败，看本文档 第 14 章 故障排查，通常是浏览器路径检测不到。

9.4 输出文件去哪了

所有生成的文件在 sop_generate 文件夹下的 output/ 子目录里：

sop_generate-win/         (或 sop_generate-mac/)
├── sop_generate.exe       (或 .app)
├── products/              （你的产品 YAML）
├── assets/images/         （你的产品图片）
└── output/                 ← 生成的 HTML 和 PDF 在这里
    ├── XESA01.html
    ├── XESA01.pdf
    └── TEST01.pdf

点工具栏没有"打开输出目录"按钮，需要自己用文件管理器去打开 output/。

10. 自动更新

10.1 工作原理

  1. 软件每次启动后约 1-2 秒，后台检查公司发版仓库是否有新版本
  2. 有新版 → 弹窗提示，你可以选 "立即更新" / "暂不更新" / "跳过此版本"
  3. 选立即更新 → 软件启动独立的 updater 子程序，自己退出
  4. updater 下载新版本分卷 → 校验 → 解压 → 替换文件（保留你的产品数据）→ 启动新版本
  5. 整个过程 2-5 分钟，进度条全程可见

10.2 升级时保留的数据

升级自动保留以下目录，不会被新版本覆盖：

目录内容
products/所有产品 YAML 文件（你的全部工序数据）
assets/images/所有产品图片
output/已经生成的 HTML/PDF（这个你可以自己定期清）

其他文件（sop_generate.exe、_internal/、updater.exe 等）会被新版本替换。

10.3 手动检查更新

如果不想等启动自动检查，可以随时点工具栏 检查更新：

  • 有新版 → 弹窗询问
  • 已是最新 → 弹窗"当前 vX.Y.Z 已是最新"
  • 网络异常 → 弹窗"无法连接，请检查网络"

10.4 跳过某个版本

升级弹窗里选 "跳过此版本" 后，软件会记一个 .skip_version 文件，启动检查时不再提示这个版本。下次再有更新（更高版本）的版本仍会提示。

11. 数据迁移（旧版本升级到新版本）

大多数时候自动更新就够了。但如果你是全新装一个新版本（比如自动更新失败、换了电脑、想保留多个版本），需要手动迁移老数据。

11.1 用"导入旧数据"按钮（推荐）

  1. 解压新版本到一个新文件夹（例：D:\sop_generate-v2.0.0\）
  2. 启动新版本 sop_generate.exe
  3. 点工具栏 导入旧数据
  4. 文件对话框里选老版本根目录（含 products/ 子目录的那一层，例：D:\sop_generate-v1.0.0\）
  5. 点确定，软件自动复制：
    • 所有 products/*.yaml
    • 所有 assets/images/<产品>/ 目录
  6. 弹窗显示"导入 N 个 YAML + M 张图片"
  7. 左侧产品列表立即刷新
同名文件不会被覆盖。比如新版本里已经有 XESA01.yaml（来自示例），从老版本导入同名 YAML 时会跳过保留新版的。如果想替换，需要先删除新版本里的同名文件再导入。

11.2 手动复制（不推荐）

如果"导入旧数据"按钮不能用，可以手工操作：

  1. 关闭新版本
  2. 用文件管理器进入老版本目录，复制 products/ 和 assets/images/ 两个文件夹
  3. 粘贴到新版本目录，提示是否覆盖时选是（你的数据覆盖示例数据）
  4. 启动新版本，左侧应该能看到老产品
千万不要用 7-Zip 直接把新版本解压覆盖到老版本目录！这样你的 products/ 和 assets/ 会被新版自带的示例数据覆盖掉。

11.3 删除老版本

新版本正常工作后，老版本目录可以安全删除。建议保留 1-2 周确认无问题再删，万一新版有 bug 还能回滚。

12. 字段约束速查表

这份表给经常用软件的工艺员速查用。所有限制都是为了 A3 单页排版不被破坏。

字段 限制 超出怎么办 建议
processes（工序总数） 1-32 道 报错 流程图按 8 个/页自动分页
operations（操作说明） 1-18 条/工序 报错 单页 6 条，超 6 自动拆页；推荐拆成多道工序
notes（注意事项） 0-4 条/工序 报错 超出请把工序拆分
tools（工具设备） 0-4 项/工序 报错 超出请把工序拆分
materials（作业材料） 0-4 项/工序 报错 超出请把工序拆分
images（图片） 1-4 张/工序 报错 1 张全占 / 2 张上下 / 3-4 张 2×2 网格
工序名 无硬限 自动换行 建议 ≤ 12 汉字
单条操作说明长度 无硬限 自动换行 建议 ≤ 30 汉字
单条注意事项长度 无硬限 自动换行 建议 ≤ 28 汉字
工具/材料单项长度 无硬限 溢出裁切 建议 ≤ 10 汉字

12.1 关于"自动拆页"

当一道工序的 operations 超过 6 条，软件自动把它拆成多个 PDF 页，每页 6 条，工序号保持不变，标记"续 N/M"。

原工序 operations 条数拆成几页
1-6 条1 页（不拆）
7-12 条2 页，标"续 1/2"、"续 2/2"
13-18 条3 页，标"续 1/3" ... "续 3/3"

拆页时图片每页都重复显示，注意事项 / 工具 / 材料只在第一页显示，续页写"见首页"。

能拆工序就别拆页。把"超长工序"拆成两道独立工序，可以让图片、注意事项、工具、材料各自配套，比续页清晰得多。

13. 常见问题（FAQ）

Q1：YAML 是什么？我能编辑吗？

YAML 是软件用来存储产品数据的文本文件，纯文本，用记事本/VS Code 能打开。普通用户不要直接编辑，请用软件 GUI。但如果想批量处理（比如全局替换某个工具名），可以用 VS Code 打开 products/<型号>.yaml 修改。

Q2：能在 Word 里继续编辑导出的 PDF 吗？

不可以。本工具的输出是固定排版的 PDF，不是 Word 文档。如果客户要 Word 格式，建议把 PDF 当附件，或者截图嵌入到客户的 Word 模板。

Q3：能改模板样式吗？比如换公司 Logo？

软件 GUI 暂不支持改模板。如果需要改：

  • 简单的颜色/字号 → 让 IT 找软件维护者改 templates/style.css.j2
  • 加 Logo / 改封面布局 → 同上，改 templates/cover.html.j2
  • 这些改动会作为新版本发布，所有用户自动接收

Q4：同一台电脑两个人轮流用，数据会混吗？

不会混，但没有用户隔离。两人共用 products/ 和 assets/images/。建议：

  • 不同人用不同产品型号前缀（例：XESA-XX-01、RX-XX-01）
  • 或者每人在自己电脑账户下装一份

Q5：升级以后旧 PDF 还能打开吗？

能。已经导出的 PDF 是独立文件，和软件没关系。升级软件不影响老 PDF。

Q6：可以批量导出多个产品的 PDF 吗？

GUI 暂时一次一个。如果想批量，让 IT 跑命令行：

cd /path/to/sop_generate
.venv/bin/python gen.py products/*.yaml --pdf

会把所有产品全部导出。

Q7：能离线使用吗？

软件本身完全离线工作。需要联网的只有两件事：

  • 启动时检查更新（网络异常会静默跳过，不影响使用）
  • 下载新版本（需要的时候才下）

编辑产品、生成 PDF 等所有核心功能都不需要网络。

Q8：导出的 PDF 是 A3 大小，公司只有 A4 打印机怎么办？

打印对话框里选 "缩放" → "适合页面" 或者 "50%"，可以把 A3 缩到 A4 打印。但字会变小。建议：

  • 能用 A3 打印机就用 A3（多数公司有彩印 A3）
  • 只能 A4 的话，分页打印：A3 横向页打印 2 张 A4 拼起来

14. 故障排查

14.1 软件启动报错

现象原因解决
"系统找不到指定文件" / 双击 .exe 闪退 解压时少了 _internal/ 文件夹 重新下载所有分卷，确认 _internal/ 在 .exe 同级
"sop_generate 已损坏，无法打开"（Mac） 苹果安全机制 系统设置 → 隐私与安全性 → 找到 sop_generate → 仍要打开
Defender 阻止 未签名应用 "更多信息 → 仍要运行"

14.2 导出 PDF 失败

错误信息原因解决
"未找到 Chrome / Edge" 本机没装支持的浏览器 装 Microsoft Edge 或 Google Chrome（任一即可）
导出后 PDF 是空白页 浏览器版本太旧 更新 Edge / Chrome 到最新版
PDF 里图片不显示 图片文件丢失或路径错 检查 assets/images/<型号>/ 下文件存在，且 YAML 里文件名拼写正确

14.3 自动更新失败

现象解决
"无法连接到 Gitee" 检查网络连接；如果公司有代理，让 IT 配置允许访问 gitee.com
升级中途网络断了 关闭 updater 窗口，重启软件，重新点更新（已下载的分卷会自动续传）
升级后软件打不开 用"导入旧数据"方式手动装新版本（见第 11 章）

14.4 工序拆页或截断异常

极少数情况会出现"工序被截断"或"PDF 底部有灰条"。如果遇到：

  1. 截图保留
  2. 记录是哪个产品、哪道工序
  3. 联系 IT 反馈
